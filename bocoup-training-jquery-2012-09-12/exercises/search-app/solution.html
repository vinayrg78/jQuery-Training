Searching should query the server and display results. (solution)


Modify exercise.js to make the exercise page work like the solution page.


Phase 1
  • When the form is submitted, send an Ajax GET request to /data/search.json that includes a q parameter whose value is the search term.
  • Using Ajax, fetch the template at /templates/people-detailed.tmpl and use the Underscore.js _.template() method to render search results.
  • Display the message No results found. if no matching results were found.
Phase 2
  • When a search result "Remove" button is clicked, that search result should be removed from the list.
  • When a search result "Like" button is clicked, the name from that search result should be added to the "Liked" list and the "You haven't liked anything yet" message should be removed.
  • Subsequent searches should work as expected: "Remove" and "Like" buttons should continue to work, the "Liked" list should persist and old search results should be removed.
Phase 3
  • Display the message Searching... while a search is pending.
  • Avoid making a request for a search that is empty or that only contains spaces.
  • Prevent the user from submitting a new search while an existing search is in progress.

If you need help, use the jQuery API documentation, Underscore.js documentation, MDN JavaScript Reference and the browser's developer tools!
